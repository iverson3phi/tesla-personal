# 애프터블로우 트리거 PWA — 설계 문서

작성일: 2026-06-29

## 배경 / 목적

기존에는 갤럭시 S22의 **MacroDroid(유료)** 가 차량 블루투스 끊김을 감지해
`POST https://ntfy.sh/tesla-ab-9f3k7q2zx8m` (본문 `afterblow`)를 보내고,
PC 데몬이 이를 구독하다가 애프터블로우(에어컨 증발기 건조)를 실행했다.

이제 MacroDroid를 유료로 쓰지 않기 위해, **부부 두 명이 쓸 보기 좋은 웹앱(PWA)** 으로
대체한다. 단순 대체를 넘어, **건조 시간을 1~10분 사이로 골라서 트리거**하고
**창문 환기(vent) 여부도 매번 선택**할 수 있게 한다.

비배포 / 비상업 / 개인용(부부 2인). Play 스토어 등록 없음.

## 범위

작업은 두 부분이다.

1. **폰 PWA** — Cloudflare Pages에 호스팅하는 단일 페이지 웹앱. 시간/환기 선택 후
   ntfy로 POST.
2. **PC 수신측 수정** — 기존 셸 스크립트 2개가 메시지에서 분/환기를 파싱해
   `tesla-sentry afterblow`로 전달하도록 변경.

비목표(YAGNI):
- 홈 화면 "진짜 위젯"(앱 안 열고 홈에서 직접 조작) — PWA로는 불가하며, 이를 위해
  네이티브 앱을 만들지 않는다. PWA 홈 아이콘 → 앱 실행 흐름으로 충분하다고 합의함.
- 차량 동작 결과 확인/피드백 — ntfy는 fire-and-forget이라 "전송 성공"까지만 표시.
- 사용자 계정/멀티유저, 설정 저장 서버 등 백엔드.

## 전체 구조

```
[폰 ×2]  PWA (Cloudflare Pages)
  │  시간/환기 선택 후 "건조 시작" 탭
  │  fetch POST  →  body: "afterblow <분> [vent]"
  ▼
[ntfy.sh]  tesla-ab-9f3k7q2zx8m  (토픽 그대로)
  │
  ▼
[PC]  afterblow-listener.sh (파싱)  →  afterblow-run.sh (분/환기 인자)
        →  tesla-sentry afterblow <분> [vent]
```

폰은 ntfy로 outbound POST만, PC는 outbound 구독만 사용하므로 공인 IP·포트개방·도메인이
필요 없다(기존과 동일).

## 메시지 포맷 (앱 ↔ PC 계약)

- 형식: `afterblow <분> [vent]`
  - 예: `afterblow 2`, `afterblow 3 vent`
- **하위 호환**: 분이 없는 단독 `afterblow`는 기본 3분으로 처리한다. 기존 동작 및
  `curl -d afterblow ...` 테스트가 그대로 동작한다.
- **분 범위**: 1–10. 앱에서 입력을 제한하고, **PC에서도 반드시 재검증**한다
  (토픽은 공개 URL 기반이라 신뢰할 수 없는 입력으로 간주):
  - 숫자가 아니면 → 기본 3분으로 처리
  - 범위를 벗어나면 → 1~10으로 클램프
- `vent` 토큰이 있으면 환기 ON, 없으면 OFF.

## 폰 PWA

### 화면 (단일 페이지)

```
   애프터블로우

   건조 시간      2분
   [───●────────]   1 ─ 10분

   창문 살짝 열기   [ ON ●]

   [      건조 시작      ]

   ─ 전송됨 ✓   (실패 시 빨강 + 재시도)
```

- Material 느낌의 깔끔한 UI, 큰 터치 영역, 다크 모드 대응, 갤럭시 S22 화면 기준.
- 시간 선택: 슬라이더 또는 +/− 스테퍼(1–10분), 현재 값 크게 표시. 기본값 3분.
- 환기: 토글 스위치("창문 살짝 열기").
- "건조 시작" 버튼 → ntfy로 `fetch` POST:
  - 본문: `afterblow <분>` + (환기 ON이면 ` vent`)
  - `Content-Type: text/plain`
- 결과 표시:
  - HTTP 200 → "전송됨 ✓"(초록)
  - 실패/네트워크 오류 → 빨강 메시지 + 재시도 버튼
  - **정직한 문구**: "전송됨"은 *메시지 전송 성공*이며 차량 동작 확인이 아님을 분명히 한다.

### 구성 파일

| 파일 | 역할 |
|---|---|
| `index.html` | 마크업 |
| `app.js` | 슬라이더/토글 상태, 버튼 클릭 시 ntfy POST, 결과 표시 |
| `style.css` | Material 느낌 스타일, 다크 대응 |
| `manifest.json` | PWA 설치 정보(이름, 아이콘, `display: standalone`, 테마색) |
| `sw.js` | 서비스워커(설치형/오프라인 실행을 위한 정적 자원 캐시) |
| 아이콘 | 단순 아이콘(192/512px) — 홈 화면 아이콘용 |

토픽 URL은 `app.js` 상단 상수 한 곳에 둔다(추후 변경 용이).

### 오류 처리

- `fetch` 실패(오프라인/타임아웃) → 사용자에게 빨강 에러 + 재시도. 전송은 본질적으로
  네트워크가 필요함을 안내.
- 서비스워커는 정적 자원만 캐시하며, POST 요청을 가로채거나 큐잉하지 않는다(과설계 방지).

## PC 쪽 수정 (기존 스크립트 2개)

### `scripts/afterblow-listener.sh`

- 현재: `if [ "$line" = "afterblow" ]` 로 정확히 일치할 때만 핸들러 호출.
- 변경: **첫 토큰이 `afterblow`인 줄**을 수신하면, 나머지 토큰(분, `vent`)을 그대로
  핸들러 인자로 넘긴다. 그 외 줄/keepalive는 무시.

### `scripts/afterblow-run.sh`

- 현재: `DURATION_MIN`/`VENT`가 스크립트 상단에 하드코딩.
- 변경: **인자로 받은 분/환기**를 사용한다.
  - `$1` = 분(없거나 숫자 아니면 기본 3, 범위 밖이면 1–10으로 클램프)
  - `$2`(또는 나머지 인자 중 `vent` 존재 여부) = 환기 ON/OFF
- **디바운스**: 기본값을 600초 → **60초**로 낮춘다. BT 깜빡임 트리거가 사라지고
  수동 트리거로 바뀌므로, 실수 더블탭만 막고 의도적 재트리거는 허용. (값은 그대로
  스크립트 상단에서 조절 가능.)
- 이후 `tesla-sentry afterblow <분> [vent]` 호출은 기존과 동일.

## 배포 · 보안

### 배포

- Cloudflare Pages에 정적 파일 배포(`wrangler pages deploy`). 이미 이 프로젝트에
  Cloudflare/Wrangler가 세팅되어 있어 그대로 활용.
- 생성된 `*.pages.dev` URL을 두 폰의 Chrome에서 "홈 화면에 추가" → 전체화면 앱처럼 사용.

### 보안

- 토픽 비밀값이 페이지 JS에 노출된다. 다만 토픽 자체가 이미 "공개 URL 기반 비밀"이므로,
  페이지 URL을 부부 둘만 알면 실질적으로 충분하다.
- (옵션, 기본 미적용) 더 잠그고 싶으면 Cloudflare Access로 페이지를 특정 계정에만 허용
  가능. 이번 범위에는 포함하지 않는다.

## 테스트

- **앱 단위**: "건조 시작" 시 브라우저 네트워크 탭에서 ntfy POST 200 확인. 오프라인 시
  에러/재시도 동작 확인.
- **PC 파서 단위**: 다음 입력에 대해 동작 검증
  - `afterblow` → 3분, 환기 OFF (하위 호환)
  - `afterblow 2` → 2분
  - `afterblow 3 vent` → 3분, 환기 ON
  - `afterblow 99` → 10분으로 클램프
  - `afterblow abc` → 3분(기본)
  - 비-afterblow 줄 → 무시
- **엔드투엔드**: 폰에서 트리거 → `afterblow.log`에 `recv` + 파싱된 분/환기 →
  `TRIGGERED (Nmin, vent=...)` 확인. 차량 동작은 차를 보면서 짧게(`tesla-sentry afterblow 1`) 확인 권장.

## 합의된 결정값

- 시간 선택 UI: 슬라이더/스테퍼 + "건조 시작" 버튼
- 분 범위: 1–10, 기본 3
- 환기: 앱 토글로 매번 선택
- 디바운스: 60초
- Cloudflare Access 잠금: 미적용(옵션으로만 문서화)
- 앱 형태: PWA(홈 아이콘 → 앱 실행). 네이티브 위젯 미사용.
